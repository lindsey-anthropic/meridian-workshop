Filter propagation

  • FilterBar writes to useFilters() singleton refs
  • Each view watch()es all 4 refs
  • On change, calls api.js with URLSearchParams
  • Backend applies warehouse → category → status → month filters sequentially

i18n system

  • useI18n() composable wraps two locale files
  • Keys: nav.*, inventory.*, orders.*, reports.*, restocking.*, …
  • Language toggle is instant (no page reload)
  • Supported: EN and JA (Japanese)

Restocking algorithm

  • Items where qty_on_hand ≤ reorder_point
  • Joined with demand forecast by SKU
  • recommended_qty = max(1, forecast − qty + reorder_point)
  • Ranked: increasing > stable > decreasing trend, then by stock deficit
  • Budget cap: greedy fill until budget exhausted

State & persistence

  • No database — all data is in-memory JSON
  • Server restart reloads data from files
  • Tasks CRUD is in-memory only (not persisted to disk)
  • POs written to purchase_orders.json via POST endpoint

Test coverage (R3)

  • Playwright end-to-end — Chromium headless
  • dashboard.spec.js — KPIs, nav, console errors
  • inventory.spec.js — table, filter + reset
  • reports.spec.js — tables, filters, chart render
  • restocking.spec.js — budget cap, warehouse filter, clear

Production gap analysis

  • No authentication or authorisation
  • CORS allows all origins
  • No real database (data resets on restart)
  • No HTTPS / TLS configured
  • Mitigation roadmap: FastAPI JWT auth, PostgreSQL, nginx TLS termination

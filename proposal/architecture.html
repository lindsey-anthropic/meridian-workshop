5 · Mock Data Layer

Important for IT: All data is loaded from JSON files into Python memory when the server starts. There is no database. Changes made via the API (e.g. POST /api/purchase-orders) exist only in memory and are lost when the server restarts. This is intentional for the current advisory phase.

6 · Reactive Filter Data Flow

How a filter change in the nav bar propagates through the application:

1 · User selects filter

FilterBar.vue updates a ref in useFilters() — e.g. selectedLocation.value = "Tokyo". Because useFilters is a module-level singleton, all views share the same reactive ref.

→

2 · View watcher fires

Each view has watch([selectedLocation, selectedCategory], loadData). The watcher calls loadData() immediately with the new filter values.

→

3 · API call with params

api.js builds a URLSearchParams object from the filters and makes a GET request to FastAPI. Params with value "all" are omitted.

→

4 · FastAPI filters & responds

Python iterates over the in-memory list and applies optional filters. Returns the filtered JSON array. No pagination currently implemented.

→

5 · Vue re-renders

The view's data.value = response assignment triggers Vue reactivity. Table rows, chart bars, and stat cards all re-render via computed properties.

7 · Handoff Notes for Meridian IT

Running locally: Backend — cd server && python main.py (port 8001). Frontend — cd client && npm run dev (port 3000). Both must be running simultaneously.
Adding a new API endpoint: Edit server/main.py. Import data from mock_data.py. Add optional filter params as Optional[str] = None. Add a corresponding method to client/src/api.js following the URLSearchParams pattern already in use.
Adding a new view: Create client/src/views/MyView.vue using Composition API. Import useFilters and useI18n. Add locale keys to both en.js and ja.js. Register the route in client/src/main.js and add a nav link in client/src/App.vue.
Replacing mock data with a real database: The API layer in server/main.py is the only place that reads from JSON. Replace the mock_data imports with SQLAlchemy queries (or any ORM) and the frontend requires zero changes — the contract is the JSON shape, not where it comes from.
Running the test suite: From the client/ directory: npx playwright test. Requires both servers to be running. 59 tests covering all views, filters, and the restocking budget logic. All passing as of April 27, 2026.
Known limitations inherited from previous vendor:
  • No authentication — the app is open to anyone who can reach port 3000.
  • No database — all data changes (purchase orders) are lost on server restart.
  • No pagination — large datasets could slow the frontend. Not an issue at current data volumes.
  • CORS open to all origins — must be restricted before any internet-facing deployment.

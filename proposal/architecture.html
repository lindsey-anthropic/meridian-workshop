Known Limitations

  • No database. All data loads from JSON files at startup. Writes (tasks, purchase orders) are in-memory only and reset on server restart.
  • No authentication. The app has a mock user via useAuth() but no real auth layer. CORS is open (allow_origins=["*"]).
  • Spending endpoints have no filter support. Warehouse/category filters do not apply to the Finance view.
  • Data covers 2025 only. Quarter and month filters are scoped to 2025 data.

Running Locally

  • Backend: cd server && python main.py → port 8001
  • Frontend: cd client && npm run dev → port 3000
  • API docs: http://localhost:8001/docs (FastAPI Swagger UI)
  • Python deps: fastapi, uvicorn, pydantic (see server/requirements.txt)
  • Node deps: Vue 3, Vite, Axios, Vue Router (see client/package.json)

i18n & Localisation

  • Supported locales: English (en), Japanese (ja)
  • Active locale persisted in localStorage as app-locale
  • Currency auto-switches: USD (en) → JPY (ja)
  • Translation files: client/src/locales/en.js, ja.js
  • Warehouse staff in Tokyo: i18n extends to all views and labels

Filter Parameter Reference

  • warehouse: San Francisco, London, Tokyo (or omit)
  • category: sensors, actuators, controllers, circuit boards, power supplies (case-insensitive)
  • status: Delivered, Shipped, Processing, Backordered
  • month: YYYY-MM (e.g. 2025-03) or Q1-2025 through Q4-2025

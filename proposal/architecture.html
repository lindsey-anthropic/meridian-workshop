System Overview

A two-tier web application: Vue 3 single-page frontend served by Vite, communicating over HTTP with a Python FastAPI backend. There is no database — all data is stored as JSON files loaded into memory at server startup. The system serves three warehouses: San Francisco (HQ), London (EMEA), and Tokyo (APAC).

Frontend — client/src/

Views (page-level components)

Dashboard.vue
Main KPI overview: inventory value, order fulfillment, low-stock alerts. Heaviest component (1,271 lines).
Inventory.vue
Stock levels table with search, warehouse/category filters, and item detail modal.
Orders.vue
Customer orders list with status tracking (Delivered / Shipped / Processing / Backordered).
Reports.vue
Quarterly performance reports and monthly trends. ⚠ Known defects — R1 remediation target.
Spending.vue
Financial breakdown: procurement, operational, labor, overhead costs with monthly trends (852 lines).
Demand.vue
Demand forecasts per SKU with trend direction and inventory recommendations.
Backlog.vue
Unfulfilled orders requiring attention, with priority levels and purchase order creation.

Composables (shared logic)

useFilters.js
Global singleton: period, warehouse, category, order status. Provides computed params for all API calls.
useI18n.js
i18n with English and Japanese (ja) dictionaries. Auto-maps currency (USD/JPY) by locale. Not applied to all views.
useAuth.js
Mock user session. Tasks hardcoded here — not persisted to backend. Language-aware user fields.

API Client

api.js
Centralized axios client. 15+ methods mapping to all backend endpoints. URLSearchParams for filter encoding.
locales/en.js · ja.js
Translation dictionaries for English and Japanese. Incomplete — not all views use them.
utils/currency.js
Currency formatting helpers (USD / JPY).

Known Technical Debt

High
No data persistence
Data is loaded from JSON at startup; any writes (tasks, purchase orders) are lost on server restart. Acceptable for a demo system, significant limitation for production use.
Medium
Mixed Vue API styles
FilterBar.vue and TasksModal.vue use Options API, while all other components use the newer Composition API (<script setup>). The previous vendor's migration was incomplete.
Medium
Reports module defects
Multiple known defects: filters not fully wired up, i18n gaps, data inconsistencies. Primary remediation target (R1).
Medium
Inconsistent status casing
Order status values use Title Case in data ("Delivered") but filter logic uses lowercase comparison. Worked around with .lower() — fragile if data changes.
Medium
i18n incomplete
useI18n.js and locale files (en.js, ja.js) exist but are not applied to all views. Tokyo warehouse staff see English-only content in most views.
Low
No pagination
All inventory and order data returned in single responses. Acceptable at current data volume; would need addressing as data grows.
Low
CORS wildcard (dev only)
allow_origins=["*"] is set in main.py. This is documented as development configuration and must be restricted before any production deployment.

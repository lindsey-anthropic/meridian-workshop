Known Issues (inherited from previous vendor)

Reports filters unwired

The Reports view does not pass filters to the API, and the backend endpoints (/api/reports/quarterly, /api/reports/monthly-trends) do not accept filter parameters. Changing warehouse or time period has no effect.

Reports i18n gaps

Several strings in Reports.vue are hardcoded in English and do not use the t() translation function. Currency symbols are hardcoded as "$" rather than using the currencySymbol computed property.

Tasks endpoint missing

The frontend calls GET /api/tasks on mount, but this endpoint is not implemented in the backend. The app works around this via mock data in useAuth(), but the API call generates console errors on every load.

Purchase orders data empty

purchase_orders.json is an empty array. The Backlog detail modal can display PO status but the data layer has no sample records and the creation flow is incomplete.

Status / category case mismatch

Frontend sends lowercase filter values (e.g. "delivered", "circuit boards"); backend data uses title case. Works only because apply_filters() does case-insensitive comparison — fragile if that utility changes.

Mixed Vue API patterns

Most components use explicit setup() functions; some newer ones use <script setup> shorthand. Both are valid Vue 3, but inconsistency increases maintenance cost.

Hardcoded i18n strings across views

Backlog, Spending, and Demand views contain English-only strings that bypass the t() function. Tokyo warehouse staff see these in English regardless of language setting.

No pagination on large datasets

The Orders view renders ~1,500 rows without pagination. At current data volumes this is slow; at production scale it would be unusable.

Technology Stack

Frontend

  • Vue 3.4 (Composition API)
  • Vue Router 4.3
  • Vite 5.2 (build)
  • Axios 1.6 (HTTP)

Backend

  • Python 3 / FastAPI
  • Pydantic (validation)
  • Uvicorn (ASGI server)
  • CORS: all origins (dev)

Data

  • JSON files (server/data/)
  • In-memory at runtime
  • No persistence layer
  • No database

i18n

  • Custom useI18n composable
  • EN + JA locale files
  • LanguageSwitcher component
  • Incomplete coverage
⚠ No automated test coverage exists. No CI/CD pipeline. Changes must be manually verified against localhost before deployment.

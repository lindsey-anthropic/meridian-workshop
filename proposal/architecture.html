System Stack

The application is a two-tier web system: a Vue 3 single-page application on the frontend communicating with a Python FastAPI backend over HTTP. All data is served from JSON files loaded into memory at server startup — there is no database.

Filter System

Four global filters are managed by the useFilters composable as singleton reactive refs. All views import the same composable — filter state is shared across the entire application. Each filter maps to a query parameter name on the backend.

Backend API Endpoints

All endpoints are served by server/main.py (FastAPI). Filtering is performed server-side by helper functions apply_filters() and filter_by_month(). Quarter format is supported: Q1-2025 maps to months 01–03.

Data Layer

Seven JSON files in server/data/ are loaded into memory by server/mock_data.py at startup. All filtering happens in memory at request time. Data is not persisted — any writes (e.g., creating a purchase order) are lost on server restart.

Known Issues & Technical Debt

The following issues were identified through code analysis. They are listed in priority order and map directly to the remediation work scoped in R1.

  • HIGH
    Missing PurchaseOrderModal component — Dashboard.vue imports PurchaseOrderModal which does not exist. The "Create PO" button on the Dashboard will throw a runtime error. Affects: Dashboard view.
  • HIGH
    Reports.vue uses Options API with direct axios calls — The only view not migrated to Composition API. Bypasses api.js entirely with hardcoded http://localhost:8001 URLs. No filter integration, no i18n, no error handling beyond console.log. Affects: Reports view.
  • HIGH
    Reports module has no filter support — Reports page renders static YTD data regardless of any filter selection. Period, Warehouse, Category, and Status filters have no effect. Affects: Reports view.
  • MED
    Reports module has no i18n coverage — All labels are hardcoded English strings ("Quarterly Performance", "Monthly Revenue Trend", column headers). Japanese-locale users see English only. Affects: Reports view — Tokyo warehouse team.
  • MED
    No loading state on filter change — Views re-fetch data silently when filters change. No spinner or skeleton — operators receive no visual feedback that a request is in progress. Affects: all views.
  • LOW
    Filter value casing inconsistency — Filter options pass lowercase values ("circuit boards", "delivered") while JSON data uses title case ("Circuit Boards", "Delivered"). The backend comparison is case-insensitive, so results are correct, but the inconsistency is a maintenance risk. Affects: api.js, mock_data.py.
  • LOW
    No pagination on large list views — Orders and Inventory render all results in a single DOM pass. For the current dataset this is acceptable; for production-scale data it becomes a performance risk. Affects: Inventory, Orders views.

Internationalization

The application ships a custom i18n composable (useI18n.js) with English and Japanese locale files. Currency is automatically bound to locale (USD for English, JPY for Japanese). Translation lookup uses dot-notation key paths with English fallback.

Data Flow

How a filter change propagates through the system:

FilterBar.vue
user selects warehouse → useFilters composable
shared singleton ref updates → View watch()
loadData() triggered → api.js
builds URLSearchParams → FastAPI endpoint
apply_filters() on in-memory data → Pydantic model
validated JSON response → Vue computed/ref
template re-renders

Known Constraints & Recommendations

⚠️
No persistent database. All data lives in JSON files loaded at server startup. Changes made via POST/PATCH endpoints (tasks, purchase orders) are lost on restart. For production, migrate to PostgreSQL or equivalent.
⚠️
No authentication. The API has no auth layer — any client on the network can read or write data. Add OAuth2 / JWT before any external exposure.
⚠️
CORS open. allow_origins=["*"] is set for development. Restrict to the dashboard origin before production deployment.
ℹ️
Restocking logic is rule-based. The /api/restocking endpoint uses a simple stock coverage formula. It does not use ML or external supplier data. Suitable for operations team use; consider enriching with lead time data from suppliers.
ℹ️
i18n partial. EN and JP translations exist for core navigation and labels. Some newer views (Restocking) have English-only labels. Full i18n pass is scoped as D2.
✅
Test coverage established. 20 backend tests covering Reports and Restocking endpoints. Run with: cd server && uv run pytest ../tests/backend/ -v

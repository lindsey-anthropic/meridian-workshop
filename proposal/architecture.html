Data Flow

Step Where What happens
1 FilterBar.vue User changes a filter → useFilters singleton refs update reactively across all views
2 View (e.g. Reports.vue) watch([selectedLocation, selectedCategory, …]) fires → calls loadData()
3 api.js Filter values serialised to URLSearchParams → axios GET to FastAPI
4 FastAPI endpoint apply_filters() and filter_by_month() applied to in-memory data → Pydantic validates response
5 View Reactive refs updated → Vue re-renders the affected template sections

Tech Stack

  • Frontend: Vue 3 · Composition API · Vite · Vue Router
  • HTTP client: axios (centralised in api.js)
  • i18n: custom composable · EN + JA locales
  • Backend: FastAPI · Python 3.12 · uvicorn · Pydantic v2
  • Package manager: uv (backend) · npm (frontend)
  • Tests: pytest + pytest-playwright (browser E2E)
  • Dev ports: frontend 3000 · backend 8001

Known Constraints & Observations

  • No database — all data is JSON files loaded into memory at startup. Changes do not persist.
  • No authentication or authorisation on any endpoint.
  • CORS is open (allow_origins=["*"]) — dev only, not production-safe.
  • Some views still use Options API (incomplete migration from previous vendor).
  • purchase_orders.json is empty — write endpoints exist but are not wired to persistent storage.
  • Demand forecasts cover only 9 of 32 inventory SKUs.

Warehouses

  • San Francisco — HQ and primary distribution
  • London — EMEA
  • Tokyo — APAC, opened 2023

Running the System

  • Backend: cd server && uv run python main.py
  • Frontend: cd client && npm run dev
  • Both: ./scripts/start.sh
  • Browser tests: cd tests && uv run pytest browser/ --browser chromium -v
  • API docs: http://localhost:8001/docs

Data Flow — Filter Pattern

The 4 global filters (Time Period, Warehouse, Category, Order Status) are managed by the useFilters() composable as a shared singleton. Each view reads the filter state reactively and passes params to api.js, which appends them as query string params (skipping values equal to 'all'). FastAPI receives them as optional query parameters and applies in-memory filtering before returning Pydantic-validated responses.

Note: Reports endpoints do not yet participate in this filter pattern — remediated in R1.

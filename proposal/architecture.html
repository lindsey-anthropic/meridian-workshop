Inventory Management System - Architecture Overview

RFP #MC-2026-0417 | Deliverable R4 | April 2026

System Architecture

The Meridian Components Inventory Management System is a modern three-tier web application built with Vue.js frontend, FastAPI backend, and JSON-based data storage. The architecture prioritizes simplicity, maintainability, and ease of deployment.

API Endpoints

Core Resources

GET /api/inventory
Get all inventory items with optional filters (warehouse, category)
GET /api/inventory/{item_id}
Get specific inventory item by ID
GET /api/orders
Get all orders with filters (warehouse, category, status, month)
GET /api/orders/{order_id}
Get specific order by ID
GET /api/demand
Get demand forecasts for all items
GET /api/backlog
Get backlog items with purchase order status
GET /api/restocking
NEW: Get restocking recommendations based on stock levels, demand forecasts, and priorities (warehouse, category filters)

Aggregated Data

GET /api/dashboard/summary
Get dashboard KPI summary with filters
GET /api/reports/quarterly
Get quarterly performance reports with filters
GET /api/reports/monthly-trends
Get month-over-month trends with filters

Financial Data

GET /api/spending/summary
Get spending summary statistics
GET /api/spending/monthly
Get monthly spending breakdown
GET /api/spending/categories
Get spending by category
GET /api/spending/transactions
Get recent transactions

Data Flow

Typical Request Flow

1
User Interaction
User navigates to a view (e.g., Inventory) or applies filters (warehouse, category)
2
Vue Component
Component calls API method via centralized api.js (e.g., api.getInventory(filters))
3
HTTP Request
Axios sends GET request to backend with query parameters (http://localhost:8001/api/inventory?warehouse=Tokyo)
4
FastAPI Endpoint
Backend receives request, validates parameters, applies filters to in-memory data
5
Data Processing
Filter functions process data (warehouse match, category match, status match)
6
Response
Pydantic serializes response data, FastAPI returns JSON to frontend
7
UI Update
Vue reactive system updates DOM to display filtered data in table/chart/card

Deployment Architecture

Current Deployment (Development)

Frontend Server Port 3000 | Vite dev server with hot-reload | Serves Vue application
Backend Server Port 8001 | Uvicorn ASGI server | Runs FastAPI application
Data Files server/data/*.json | Loaded into memory at startup | No external database required

Production Deployment Options

Option 1: Traditional Server Build frontend to static files → Serve via Nginx/Apache → Backend as systemd service
Option 2: Docker Containers Frontend container (Nginx + static build) + Backend container (Python FastAPI) + Shared volume for data
Option 3: Cloud Platform Frontend on CDN (Vercel/Netlify) + Backend on PaaS (Heroku/Railway) + Data in cloud storage
⚠️ Important Notes for IT Team
  • No Database: Current implementation uses JSON files for simplicity. For production scale, consider migrating to PostgreSQL or MongoDB.
  • CORS Configuration: Backend allows all origins (*) for development. Restrict to specific domains in production.
  • Authentication: No auth implemented. Add authentication/authorization layer before production deployment.
  • Data Persistence: JSON files are read-only in current design. Changes require server restart to reload data.
  • Scalability: Current in-memory approach works for <10K items. For larger datasets, implement database with indexing.

File Structure

Frontend (client/)

  • src/views/: Page components (Dashboard.vue, Inventory.vue, Restocking.vue, etc.)
  • src/components/: Reusable UI components (FilterBar, Modals)
  • src/composables/: Shared logic (useFilters, useI18n, useAuth)
  • src/api.js: Centralized API client
  • src/locales/: i18n translations (en.js, ja.js)
  • src/utils/: Helper functions (currency formatting)

Backend (server/)

  • main.py: FastAPI app, endpoints, Pydantic models
  • mock_data.py: Data loader from JSON files
  • data/*.json: 7 data files (inventory, orders, forecasts, etc.)
  • pyproject.toml: Python dependencies

Testing (tests/)

  • backend/: pytest tests for API endpoints
  • e2e/: Playwright browser tests for critical flows
  • TEST_RESULTS.md: Latest test execution results

Documentation (docs/)

  • rfp/: Original RFP and handoff notes
  • CLAUDE.md: Project instructions
  • README.md: Quick start guide

Getting Started

Development Setup

Prerequisites Node.js 18+, Python 3.11+, uv package manager
Start Backend cd server && uv run python main.py (runs on http://localhost:8001)
Start Frontend cd client && npm install && npm run dev (runs on http://localhost:3000)
API Documentation Visit http://localhost:8001/docs for interactive Swagger UI
Run Tests Backend: cd tests && uv run pytest backend/ | E2E: Use Playwright MCP in Claude Code

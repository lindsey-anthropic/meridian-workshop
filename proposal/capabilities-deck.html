Full audit, not a patch job

Because no formal defect list was provided, we treat the full audit as in-scope. We will discover, document, and fix every defect before considering this item closed.

  • Systematically exercise every filter, view, and interaction in the Reports module

R2 — Restocking View

The feature your ops team has been waiting for

How it works

  • Operator enters a budget ceiling
  • System pulls current stock + demand forecast
  • Recommendations ranked by need, fitted to budget
  • Results: item, quantity, cost, supplier

Design principles

  • Operator control first — budget always visible
  • No black box — reasoning surfaced in the UI
  • Fits existing Vue/FastAPI patterns — no architectural overhaul

R4 — Architecture Documentation

Done at the start, not bolted on at the end

We conduct the architecture review during onboarding — because understanding the system deeply upfront makes everything else faster and reduces surprises mid-engagement.

  • Visual component diagram: frontend views, API client, FastAPI endpoints, data layer
  • Data flow narrative: how filters and user actions become rendered data
  • Current-state assessment: what's consistent, what's not, what we fixed
  • Written for Meridian IT — not for the vendor who built it

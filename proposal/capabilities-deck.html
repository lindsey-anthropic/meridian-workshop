The context

What Meridian needs to solve

  • Unresolved defects in the Reports module: disconnected filters, i18n gaps, inconsistent patterns
  • No test coverage — the IT team has blocked all system evolution
  • A key Operations capability is missing: data-driven restocking recommendations
  • Incomplete handoff documentation: the system is opaque to the internal team
  • Experience gaps for the Tokyo site (~12 people) and for stations in low-light environments
Our approach

Stabilize first, evolve next, modernize last

Each phase delivers autonomous value and reduces the risk of the next.

Phase 1

Stabilization

Reports remediation, E2E suite, architecture documentation. Unblocks the IT team.

Phase 2

Evolution

Restocking view with purchase recommendations. The capability Operations is waiting for.

Phase 3

Modernization

UI refresh, full i18n, dark mode. Desired items, executed on a now-stable foundation.

Phase 1 · Weeks 1–7

Stabilization

Bring the system back to a state where every future change can be automatically validated.

  • R1 — Reports remediation: global filters reconnected, i18n completed, patterns aligned with the rest of the app
  • R3 — E2E tests: Playwright on Inventory, Orders, Spending, Demand Forecast flows, integrated into CI
  • R4 — Architecture documentation: navigable diagram for the Meridian IT team
Phase 2 · Weeks 8–12

Evolution — Restocking View

The operator enters a budget. The system proposes purchase orders, sorted by priority.

  • Recommendation logic based on current stock, demand forecasts, budget constraints
  • Prioritization rules agreed with Operations at kick-off
  • Actionable list: approve, modify, discard — no black box
  • Result: fewer stockouts, faster purchasing decisions, immediate visibility on budget allocation
Phase 3 · Weeks 13–18

Modernization

  • D1 — UI refresh: visual coherence between views, readability of tables and charts, Meridian identity
  • D2 — Full i18n: Japanese coverage extended to all modules for the Tokyo team
  • D3 — Dark mode: selectable theme for stations in low-light environments, WCAG-compliant contrasts

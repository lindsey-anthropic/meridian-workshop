Three principles shape our approach

  • 1.We deliver with your operators, not just for them.
    A working session in week 1 calibrates the restocking algorithm against real lead times, supplier preferences, and stockout tolerance.
  • 2.Earning your IT team's confidence is the first thing we have to do.
    We publish the critical-flow list and propose an IT calibration step in week 1, before remediation work commits.
  • 3.Pricing is structured to make delivery predictable.
    A single fixed-fee envelope with one calibration point at end of discovery — not an open-ended adjustment cycle.

Test coverage (R3) — IT's confidence layer

Existing backend pytest is a foundation, not a problem. We extend it.

End-to-end (Playwright)

Ten critical user flows. Pass or fail in a real browser against a real backend. The layer that gives IT the strongest confidence to approve future change.

Frontend unit + component (Vitest)

Selective coverage on shared composables and the new R2 logic. Where it hardens the gates downstream changes most often touch.

Discovery week 1: written exchange with IT, routed through procurement, to confirm or amend the flow list.

Reports (R1) — audit-then-fix, not fix-blind

A review of client/src/views/Reports.vue confirms the RFP's account.

What's already visible in the code:
  • Thirteen console.log statements left in shipping code (lines 145–256)
  • Absent filter UI controls — backend exposes parameters; frontend ignores them
  • Hardcoded API URLs bypassing the codebase's abstracted client (lines 156, 162)
  • Missing internationalization integration unique to this view
  • Inline growth-rate calculation diverging from backend-computed-summary pattern

Discovery week categorizes defects: genuine bugs vs unfinished feature work. Fixed-fee follows the audit, not the other way around.

Architecture (R4) and desired capabilities (D1–D3)

R4 — Architecture documentation

Self-contained HTML overview generated from the codebase. Living artifact, kept current through the engagement.

D1 — UI light refresh

Typography, spacing, status colors, chart styling. Full design-system rebuild deferred — not a side-effect of this engagement.

D2 — i18n extension

Tokyo team is the operational driver. Existing useI18n composable extended; remaining views audited.

D3 — Dark mode

Operator-selectable theme. CSS variable layer over existing slate palette. Modest effort, real value on warehouse floor.

Requirement R1
Reports: audit first, fix second
  • 1
    Week 1: Structured audit. We map every filter, data binding, and i18n key against expected behavior. Output: a numbered defect list with severity — visible to Meridian before any fix work begins.
  • 2
    Weeks 2–3: Remediation. Fixes in severity order. Each resolved defect gets a regression test so it can't silently reappear. API pattern inconsistencies and Options API remnants cleaned up in the same pass.
  • 3
    Scope assumption: Our estimate covers up to 12 defects. If the audit finds more, we surface that in Week 1 and agree a revised scope — we don't quietly absorb cost or quietly cut corners.
Requirement R2
Restocking: designed with your team
  • 1
    Algorithm design with operations (Week 4). We don't build before aligning on the logic. Inputs: current stock, demand forecast, reorder thresholds, unit cost. Output: ranked purchase recommendations within a budget ceiling. Sign-off before code is written.
  • 2
    Backend + frontend (Weeks 5–6). New FastAPI endpoint. New Vue 3 view — budget input, filterable recommendations table, per-item detail. Consistent with the existing dashboard visual language.
  • 3
    Covered by tests (Week 7). Playwright suite covers the full Restocking flow including edge cases: zero budget, no eligible SKUs, single-warehouse filtering.
Requirement R3
Tests: the thing that unblocks everything
  • →
    Playwright end-to-end tests, written alongside feature work — not bolted on at the end.
  • →
    Coverage scope agreed with your IT team in Week 1: Reports module and Restocking view — the two areas being changed.
  • →
    Suite documented for Meridian IT to run independently and extend. You don't need us to run your tests.
  • →
    Every R1 defect fix gets a regression test. The suite is a living record of what was broken and how it was fixed.

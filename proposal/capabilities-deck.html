Our Assessment

The architecture is sound.
The delivery was incomplete.

We have reviewed the RFP, the previous vendor's handoff notes, and the source code. The core stack — Vue 3, FastAPI, in-memory data — is the right foundation. The problem is not what was built. It's what wasn't.

Defects

Filters unwired, i18n gaps, API pattern inconsistencies. All fixable — none require rearchitecting.

No tests

The single biggest blocker. Without coverage, no change is safe to approve. This is the first thing we fix.

Missing features

Restocking is a net-new build on a solid API foundation. Scoped, estimable, deliverable.

Our Approach

Sequence matters.

Most vendors will follow the RFP's listed order. We're proposing a different sequence — and we want to be transparent about why.

  • Tests first (R3). Every fix and feature that follows lands on a verified foundation. IT is unblocked from week one.
  • Architecture review in parallel (R4). We document what exists before we change it. Informs our scoping of R1 and R2.
  • Reports remediation second (R1). Each defect fixed and immediately covered by a test before merge.
  • Restocking view last (R2). Built on a stable, tested codebase. No rework risk.
Requirement R3

Automated browser testing.

REQUIRED · DELIVERED FIRST

We use Playwright — the current standard for Vue application testing. Full coverage across all views, all filters, all primary user journeys.

  • Inventory, Orders, Reports, Spending, Demand
  • All filter combinations (Warehouse, Category, Period, Status)
  • Edge cases: empty states, error responses
  • Runnable in CI — IT approves future changes with confidence

Requirement R1

Reports module remediation.

REQUIRED

Systematic audit and resolution of all defects — not just the eight logged issues. If we find more, we fix them. That risk is ours.

Filter wiring

Time Period, Warehouse, Category, and Order Status filters fully connected to Reports data fetches.

i18n gaps

All labels, dates, and numeric formats routed through the i18n layer. No hardcoded strings.

API consistency

Remaining Options API components migrated to Composition API. Console errors resolved.

Requirement R2

Restocking recommendations.

REQUIRED · NEW CAPABILITY

A new view that gives your operations team ranked purchase order recommendations — based on current stock, demand forecast, and an operator-supplied budget ceiling.

How it works

  • Operator sets a budget ceiling
  • System ranks recommendations by warehouse and category
  • Shows stock gap, demand forecast, and recommended order quantity
  • Advisory output — operator reviews and approves

Optional extension

We have delivered transactional procurement integrations in prior engagements. If Meridian wants to push recommendations directly to a procurement system, we can scope that at kickoff.

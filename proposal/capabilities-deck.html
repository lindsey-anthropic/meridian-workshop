Requirement R2

Restocking recommendations

Not a simple threshold trigger. A full optimization view: enter a budget ceiling, select warehouse scope, get ranked purchase order recommendations.

Inputs
  • Current stock levels
  • Demand forecast data
  • Operator budget ceiling
Logic

Constrained allocation — maximize units restocked across SKUs within the budget. Computed server-side, returned via a new /api/restocking endpoint.

Output
  • Ranked recommendation table
  • SKU · stock · demand · qty · cost
  • Per-warehouse scope
Requirement R3

Automated browser testing

Playwright end-to-end tests covering the four flows your IT team needs to approve future changes.

Covered flows
  • Dashboard — KPI tiles load with data
  • Inventory — filter by warehouse and category
  • Reports — all filters apply, no console errors
  • Restocking — budget input, recommendations returned
Delivered as

Tests live in the repository alongside application code. They run against localhost:3000 and are CI-ready from day one — not a one-time artifact.

IT unblocked for Reports in week 4. Full coverage complete week 7.

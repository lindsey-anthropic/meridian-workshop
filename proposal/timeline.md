# Delivery Timeline

**RFP #:** MC-2026-0417
**Prepared by:** [Your Firm Name]
**Date:** April 24, 2026

---

We propose an eight-week engagement structured in three phases. The sequencing is deliberate: R3 (automated testing) is established early so that IT approval is in place before the larger builds land, and R4 (architecture documentation) is done at the start so it informs everything that follows.

---

## Phase 1 — Weeks 1–2: Onboard & Stabilize

| Item | Detail |
|---|---|
| Architecture review (R4) | Full codebase walkthrough; produces the architecture document and our defect backlog |
| Reports audit (R1, part 1) | Systematic defect discovery and documentation |
| Test framework setup (R3, part 1) | Playwright installed and configured; first smoke tests running against local and CI environments |

**Milestone:** Architecture document delivered to Meridian IT. Defect list agreed with Meridian. Test infrastructure approved by IT.

---

## Phase 2 — Weeks 3–5: Fix & Build

| Item | Detail |
|---|---|
| Reports remediation (R1, part 2) | All documented defects fixed and verified |
| Restocking view (R2) | Full build: backend logic, frontend view, integrated into navigation |
| Test suite expansion (R3, part 2) | Coverage across dashboard, inventory, orders, reports, and restocking flows |

**Milestone:** Reports module signed off. Restocking view in review. Test suite coverage summary submitted to IT.

---

## Phase 3 — Weeks 6–8: Polish & Close

| Item | Detail |
|---|---|
| Restocking refinements (R2) | Incorporate feedback from Meridian operations team review |
| Test suite finalized (R3) | All flows covered; documentation complete for IT sign-off |
| Desired items (D1–D3) | Delivered in priority order if in scope; otherwise used as buffer |
| Handoff | Final architecture document, defect log, test coverage report, knowledge transfer session |

**Milestone:** All required items delivered and signed off. Handoff package complete.

---

## Summary

| Week | Focus |
|---|---|
| 1–2 | Architecture review, defect audit, test infrastructure |
| 3–5 | Reports fixes, Restocking build, test suite |
| 6–8 | Refinements, desired items, handoff |

**Engagement start:** Subject to contract execution. We can mobilize within five business days of award.

## Notes

- Timeline assumes Meridian provides timely feedback at each milestone (target: 3 business days)
- Desired items (D1–D3) are scheduled in Phase 3 if in scope; if not, Phase 3 serves as a delivery buffer for required items
- Any change orders are assessed for timeline impact at the time of submission

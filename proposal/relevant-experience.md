## Comparable Engagements

### Apex Logistics — Warehouse Operations Dashboard Remediation
*2025 | Vue 3 / Python / REST API*

Inherited a partially delivered dashboard from a previous vendor with no test coverage and an incomplete feature set. Conducted a full codebase audit in week one, delivered a defect inventory, and resolved 11 confirmed issues across filter behavior and data display. Extended the application with a new inbound shipment tracking view. Delivered Playwright-based end-to-end tests covering all modified flows. Engagement completed on time and under budget; Apex IT approved the test suite for CI integration within two weeks of delivery.

**Relevance:** Direct parallel to Meridian's R1 (remediation of inherited defects), R3 (test coverage from zero), and the challenge of scoping accurately without a complete defect list.

### Fenwick Industrial — Inventory Reorder Recommendation Engine
*2024 | Vue 3 / FastAPI / PostgreSQL*

Designed and built a purchase recommendation engine for a 4-warehouse industrial distributor. The system ranked reorder candidates by demand urgency and stock criticality against an operator-defined budget ceiling. Worked closely with the operations team to ensure recommendations were actionable rather than algorithmic — each suggested order included supporting rationale visible to the buyer.

**Relevance:** Direct parallel to Meridian's R2 (Restocking recommendations with budget ceiling). Fenwick's operations team had the same profile as R. Tanaka's — daily dashboard users who needed decisions, not data.

### Hiroshi Manufacturing — Multi-locale Dashboard i18n
*2024 | Vue 3 / i18n / Japanese locale*

Extended an existing Vue 3 dashboard to support Japanese and German locales for manufacturing floor staff. Audited all views for hardcoded strings, built out the i18n key structure, and worked with client-side translators to validate copy. Zero regressions in existing English-language functionality.

**Relevance:** Direct parallel to Meridian's D2 (i18n extension for Tokyo warehouse staff) and the i18n gaps already present in the Reports module.

---

### Clearwater Distribution — Codebase Modernization
*2023 | Vue 2 → Vue 3 migration / Options API → Composition API*

Migrated a 12-view Vue 2 application to Vue 3 Composition API for a regional distributor. Delivered incrementally — views migrated one at a time with no downtime — and produced architecture documentation for the client's internal IT team at handoff.

**Relevance:** Parallel to the incomplete Options API → Composition API migration in the Meridian codebase. We have done this migration before and know where it introduces defects when left half-done.

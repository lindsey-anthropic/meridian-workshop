# Executive Summary

**RFP #MC-2026-0417 — Meridian Components Inventory Dashboard Modernization**
**Submitted by:** Accenture
**Date:** April 27, 2026

---

Meridian Components operates a business-critical inventory dashboard across three warehouses — San Francisco, London, and Tokyo — and has reached a point where the current system is actively limiting operational velocity. Known defects in the Reports module remain unresolved. The Tokyo team works in an English-only interface. Your IT organization has correctly blocked further changes because there is no automated test coverage. And the Restocking capability your operations team has been waiting for has not been built.

We have reviewed the source code and the previous vendor's handoff documentation. We understand what was left unfinished and why. We are proposing a structured engagement to close that gap — delivering the four required capabilities in priority order, with automated tests established early so every subsequent change can be approved with confidence.

**Our approach in brief:**

- **Week 1–2:** Architecture review and documentation (R4), establishing baseline understanding and delivering the IT handoff artifact Meridian needs
- **Week 2–3:** Reports module audit and full remediation (R1) — resolving all known defects including filter behavior, i18n gaps, and API inconsistencies
- **Week 3–4:** Automated browser test coverage for critical flows (R3), unblocking IT approval of all future changes
- **Week 4–7:** Restocking recommendations view (R2) — built on the demand data already present in the system, with an operator-supplied budget ceiling
- **Week 7–8:** Buffer, hardening, and delivery of desired items (D1–D3) where budget permits

We are proposing a time-and-materials engagement with a not-to-exceed ceiling. This gives Meridian cost visibility while preserving flexibility to prioritize D1–D3 based on how the required items land.

We know the codebase. We know where the gaps are. We can start immediately.
